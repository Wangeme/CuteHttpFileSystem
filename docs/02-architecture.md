# 架构设计说明

## 1. 架构风格

采用端口与适配器（Hexagonal Architecture）思想，依赖方向始终指向领域内核：

```text
桌面 GUI ─┐
HTTP API ─┼─> Application/FileService ─> 安全路径、权限、文件系统
CLI ──────┘                  ├─────────> 会话服务
                             ├─────────> 可恢复上传管理器
                             └─────────> 审计端口
```

HTTP 层只完成协议解析、身份提取和错误映射，不自行拼接磁盘路径或决定权限。

## 2. 模块职责

| 模块 | 职责 |
| --- | --- |
| `config` | 读取、规范化并校验不可变运行配置。 |
| `security` | 权限模型、密码散列、会话、地址过滤。 |
| `paths` | 用户相对路径到共享目录内真实路径的唯一安全通道。 |
| `services` | 文件用例编排与领域错误，不依赖 HTTP。 |
| `uploads` | 上传会话、偏移控制、分块校验、恢复索引与原子提交。 |
| `transfers` | 线程安全地统计下载字节、速度和短期完成状态，供桌面控制台观察。 |
| `audit` | 结构化审计事件与容错写入。 |
| `http` | ASGI 路由、请求/响应转换、统一错误处理。 |
| `cli` | 进程入口、信号交由 Uvicorn 管理。 |

## 3. 关键设计决策

- 上传先写同目录临时文件，刷新并原子替换，避免中断产生可见半文件。
- 浏览器默认按 16 MiB 分块并预读下一块。服务端只接受当前精确偏移，降低请求与浏览器哈希开销。
- 恢复键只保存在浏览器本机；路径、大小、修改时间和随机恢复键共同定位会话。
- 完成阶段检查总长度、执行 `fsync` 和原子替换，并返回服务端流式计算的整文件 SHA-256；严格 API 客户端可选分块与清单 SHA-256。
- 下载使用 ASGI 文件响应提供零拷贝友好的流式发送和标准 Range 语义，不把完整文件载入内存。
- 下载跟踪响应只观察 ASGI 实际发送的 body 字节，不改变 FileResponse 的 Range 与流式行为。
- 令牌只在内存保存；进程退出即失效，适合临时共享场景，也降低落盘泄露风险。
- 账户密码采用 PBKDF2-HMAC-SHA256，每个密码独立盐值并恒定时间比较。
- 默认开放模式给局域网浏览器读、写、删除权限；账户与会话是可选受控模式，不是客户端前置条件。
- 地址规则在进入业务路由前执行；拒绝列表优先，配置允许列表后默认拒绝未命中地址。
- 符号链接的最终路径也必须位于共享根目录，阻止通过 junction/symlink 逃逸。

## 4. 演进接口

GUI 将直接管理 `ServerController`（后续里程碑），通过线程安全命令启动/停止 Uvicorn；
配置界面只生成 `AppConfig`，不直接修改正在运行的服务对象。

## 5. 可恢复上传时序

```text
浏览器                 HTTP 适配器             上传管理器/文件系统
  | POST 创建/恢复会话      |                         |
  |----------------------->| 校验路径、权限、大小      |
  |<-----------------------| upload_id/offset/chunk    |
  | PATCH offset + 块摘要   |                         |
  |----------------------->| 摘要正确后追加临时文件     |
  |<-----------------------| 新 offset/前缀清单         |
  | POST complete + 清单    |                         |
  |----------------------->| fsync、整文件 SHA、replace |
  |<-----------------------| 最终大小和 SHA-256         |
```
