# 安全威胁模型

## 1. 保护资产

共享根目录内文件、共享根目录外主机文件、账户凭据、会话令牌、审计记录和服务可用性。

## 2. 信任边界与主要威胁

| 威胁 | 控制措施 | 验证方式 |
| --- | --- | --- |
| `../`、绝对路径、编码路径穿越 | 统一解码后由 `SafePathResolver` 解析并检查公共路径。 | 单元测试覆盖父路径、盘符和绝对路径。 |
| 符号链接/目录联接越界 | 对最终解析路径执行 `relative_to(root)` 检查。 | 创建指向根外的符号链接测试（平台允许时）。 |
| 未授权读写删 | 用例入口强制检查 `Permission`。 | 每项操作均做拒绝测试。 |
| 密码库泄露 | 带盐 PBKDF2，配置不接受明文密码字段。 | 散列格式和错误密码测试。 |
| 令牌猜测与会话固定 | `secrets.token_urlsafe(32)`，登录总是创建新令牌，限时过期。 | 会话生命周期测试。 |
| 磁盘耗尽 | 配置单文件上限，上传按块计数，异常时清理临时文件。 | 超限上传测试。 |
| 半文件与覆盖竞态 | 同目录临时文件 + `os.replace` 原子提交。 | 失败清理与覆盖测试。 |
| 传输中静默损坏 | 每块 SHA-256、前缀清单和最终整文件 SHA-256；不匹配返回 422。 | 错误摘要不得推进偏移的测试。 |
| 重放或乱序分块 | 每次 PATCH 必须携带并匹配服务端当前精确偏移。 | 偏移冲突测试。 |
| 断线遗留临时文件 | 随机隐藏临时名、显式取消和 24 小时过期清理。 | 取消与清理测试。 |
| IP 伪造 | 仅信任 ASGI 连接地址；默认不采信 `X-Forwarded-For`。 | 地址过滤测试。 |
| 审计日志注入 | 每条事件 JSON 序列化，禁止手工拼接行。 | 特殊字符日志测试。 |

## 3. 已知限制

- 系统支持配置 TLS 证书与私钥，但不负责自动申请、续期或建立公网信任链。
- 内存会话不支持多进程共享；首期固定单进程运行。
- 文件系统 ACL 不由 CHFS 改写，运行账户仍必须遵守最小权限原则。
- SHA-256 能发现传输或存储损坏，但免登录开放模式不提供发送者身份保证；仅应在可信局域网使用。
