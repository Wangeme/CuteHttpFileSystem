# 软件需求规格说明书（SRS）

## 1. 目标与范围

CHFS 用于个人或小团队在可信或半可信网络中快速共享文件。系统必须在易用性与安全性之间取得平衡，
并提供稳定的内核接口，使 HTTP 服务、命令行和桌面 GUI 可以复用业务能力。

首期不包含互联网穿透、云存储同步、分布式存储和在线协作编辑。

## 2. 角色

| 角色 | 说明 |
| --- | --- |
| 局域网用户 | 默认无需登录，直接通过浏览器读写共享目录；能力由 `guest_permissions` 配置。 |
| 账户用户 | 使用用户名和密码登录，获得账户配置的权限。 |
| 管理员 | 拥有 `admin` 权限，可在后续 GUI 中管理运行参数与账户。 |

## 3. 功能需求

| 编号 | 需求 | 验收条件 |
| --- | --- | --- |
| FR-01 | 浏览目录 | 返回名称、类型、大小、修改时间，不泄露共享根目录外的信息。 |
| FR-02 | 下载文件 | 具有读权限时可下载文件；目录和越界路径被拒绝。 |
| FR-03 | 上传文件 | 具有写权限时可上传；超限拒绝；成功后不得出现半文件。 |
| FR-04 | 新建目录 | 具有写权限时可递归创建目录。 |
| FR-05 | 删除 | 具有删除权限时可删除文件；递归删除必须显式声明。 |
| FR-06 | 认证会话 | 正确凭据签发限时令牌；错误凭据不暴露账户是否存在。 |
| FR-07 | 权限控制 | 每项操作在领域服务入口检查权限，不能只依赖界面隐藏。 |
| FR-08 | 地址过滤 | 支持 IPv4/IPv6 CIDR；拒绝规则优先于允许规则。 |
| FR-09 | 审计日志 | 安全相关操作以结构化格式记录，日志失败不得破坏文件操作。 |
| FR-10 | 配置管理 | 启动前校验端口、大小、网络规则、账户和目录配置。 |
| FR-11 | 免安装客户端 | 除服务端外，局域网机器只需现代浏览器，无需安装软件或强制登录。 |
| FR-12 | HTTPS | 配置成对证书与私钥后使用 HTTPS，并生成正确访问地址。 |
| FR-13 | 大文件断点上传 | 浏览器按块上传；断线重连后从服务端已确认偏移继续，不重复提交完整文件。 |
| FR-14 | 传输完整性 | 默认快速模式校验精确偏移与总长度并返回服务端整文件 SHA-256；严格 API 客户端可附加分块和清单 SHA-256。 |
| FR-15 | 断点下载 | 下载响应支持标准 HTTP Range，客户端可恢复中断的下载。 |

## 4. 非功能需求

- NFR-01 安全：所有用户路径必须经过同一个安全解析器；密码不得明文保存。
- NFR-02 可测试：领域层不依赖网络服务器，可用临时目录执行确定性测试。
- NFR-03 可维护：模块职责单一，公共对象带类型标注和中文说明。
- NFR-04 兼容：支持 Python 3.11 及以上；首要运行平台为 Windows。
- NFR-05 可观测：HTTP 请求包含请求标识，关键动作进入审计日志。
- NFR-06 性能：上传以 16 MiB 块流式写盘并预读下一块，内存占用不随文件大小增长；下载交给 ASGI 文件响应处理。
- NFR-07 完整性：错误分块不得推进服务端偏移；最终文件仅在同步并校验完成后原子可见。
- NFR-08 容量：默认单文件上限为 1 TiB，且可通过配置下调；测试必须覆盖多分块和大文件路径。
- NFR-09 吞吐：软件层避免人为限速，并通过真实 HTTP 回环基准发现应用瓶颈；实际局域网速率受网卡、磁盘、Wi-Fi、交换机及终端浏览器共同限制。

## 5. 里程碑与质量门

1. M1 需求与架构基线：文档评审完成，模块边界明确。
2. M2 内核：领域单元测试全部通过，路径越界与权限绕过测试通过。
3. M3 HTTP：API 集成测试通过，上传、下载、认证形成闭环。
4. M4 GUI：主要任务流可用，视觉与可访问性验收通过。
5. M5 发布：打包、用户手册、回归测试与发布清单完成。
