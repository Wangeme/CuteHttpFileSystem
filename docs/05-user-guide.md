# 用户使用指南

## 1. 首次运行

1. 运行 `python -m chfs.cli gui`；桌面版会在 `%LOCALAPPDATA%\CHFS\config.json` 创建当前用户配置。
2. 默认共享目录是当前用户“下载”目录下的 `CHFShare`。
3. 默认允许局域网浏览器浏览、上传、下载和删除，无需登录。
4. 只有希望限制访问时，才需要在“账户管理”添加账户并收紧访客权限。
5. 默认监听 `0.0.0.0`，局域网设备可使用界面列出的局域网地址访问；仅本机使用时可改为 `127.0.0.1`。
6. 软件打开后会自动启动服务，可在“运行概览”复制访问地址。

如需 HTTPS，在“安全状态”同时选择 PEM 格式证书和私钥；保存并重启后访问地址会改为 `https://`。

## 2. 浏览器操作

- 点击文件夹进入目录，使用面包屑返回上级。
- 默认可选择或拖放多个文件上传，并可新建文件夹。
- 手机系统文件选择器支持一次选择多个文件；上传浮层显示文件序号、当前进度、总进度和实时速度。
- 大文件自动按 16 MiB 分块；网络中断后再次选择同一个文件，会从已确认位置继续。
- 默认快速模式不在浏览器逐块计算 SHA-256；服务端仍流式计算整文件 SHA-256，并结合严格偏移、总长度、`fsync` 和原子发布避免半文件与缺失数据。
- 下载支持浏览器/下载器的 Range 续传；最终完整性可用响应中的 SHA-256 与本地工具比对。
- 具有删除权限时才显示删除按钮；目录删除会明确提示包含其中所有内容。
- 若服务端收紧访客权限，相关按钮会禁用；配置账户后才显示登录入口。

## 3. 安全建议

- 默认监听 `0.0.0.0` 以便局域网访问；请仅在可信局域网使用，或通过允许/拒绝网段限制来源。
- 不要把未启用 HTTPS 的服务直接映射到互联网。
- 为每位使用者创建独立账户，只授予必要权限。
- 定期查看“操作日志”，确认登录、上传、下载和删除行为符合预期。
- 操作日志与审计日志是同一份记录：界面提供便于阅读的表格，磁盘上使用一行一个 JSON
  对象的 `.jsonl` 文件，便于持续追加、故障恢复和后续自动分析。“用记事本打开”无需设置
  Windows 文件关联。
- 使用操作系统普通账户运行 CHFS，不要让共享目录包含系统文件或私密凭据。
- 免登录意味着同一网络内能访问端口的设备都可按访客权限操作；不可信网络应收紧访客权限或启用 HTTPS 与账户。
- “开放全盘访客访问”会暴露运行账户可访问的所有磁盘并授予访客读、写、删除权限；只能在完全可信的隔离网络临时开启。

## 4. 性能与大文件建议

- 有线千兆局域网优先；共享目录和接收端使用 SSD 可减少磁盘瓶颈。
- Windows 防火墙只放行所选专用网络与服务端口，不要做公网端口映射。
- 默认单文件上限为 1 TiB，可通过 `max_upload_bytes` 下调。
- “跑满带宽”取决于服务端磁盘、客户端磁盘、网卡、交换机/Wi-Fi 和浏览器；CHFS 不主动限速，建议用 `scripts/benchmark_transfer.py` 在目标机器实测。

## 5. 服务端观察传输

- “传输会话”页每 0.5 秒自动刷新，显示方向、文件、来源 IP、字节进度、平均速度和状态。
- 尚未继续发送下一块的上传显示为“等待续传”；正在接收分块时显示为“上传中”。
- 已完成下载保留 10 秒，便于管理员确认刚刚由哪台设备下载了什么文件。
- 运行概览优先列出真实局域网地址；地址列表可滚动查看 IPv6，悬停地址会更新右侧二维码。
- 状态为“运行中”时按钮为红色“停止服务”；状态为“已关闭”时按钮为绿色“启动服务”。

## 6. 命令行

```powershell
$env:PYTHONPATH = "src"
python -m chfs.cli check-config --config config.json
python -m chfs.cli hash-password
python -m chfs.cli serve --config config.json
python -m chfs.cli gui --config config.json
```
